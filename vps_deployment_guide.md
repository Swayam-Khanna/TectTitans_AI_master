# VPS Production Deployment Guide

This guide details the step-by-step process of deploying the **TechTitans AI** monorepo application to an Ubuntu-based VPS (Virtual Private Server) instead of using Vercel.

## Deployment Architecture

We host the application using a classic high-performance production stack:
- **Frontend**: Vite React statically built, served directly by **Nginx** (highly optimized for static assets).
- **Backend**: Express API server managed by **PM2** process manager (running on port `8080`).
- **Database**: PostgreSQL (Supabase, Neon, or self-hosted).
- **SSL**: HTTPS configured via Let's Encrypt (Certbot).

---

## Step 1: Initial VPS Server Setup

Connect to your VPS server via SSH:
```bash
ssh root@your_server_ip
```

Update system package indexes and install Nginx, Git, and build tools:
```bash
sudo apt update
sudo apt upgrade -y
sudo apt install -y nginx git build-essential curl
```

---

## Step 2: Install Node.js, pnpm, and PM2

Install **Node.js 20 LTS** using NodeSource:
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

Install **pnpm** (used for monorepo package management):
```bash
npm install -g pnpm
```

Install **PM2** (to keep the Node backend running forever in the background):
```bash
npm install -g pm2
```

---

## Step 3: Clone Code and Set Up Directories

Create the directory for your website, configure permissions, and clone your repository:
```bash
sudo mkdir -p /var/www/TechTitans-AI
sudo chown -R $USER:$USER /var/www/TechTitans-AI

# Clone the repository directly
git clone https://github.com/techtitansai/TechTitans-AI.git /var/www/TechTitans-AI
```

---

## Step 4: Configure Production Environment Variables

Navigate to the project root and create your production `.env` file:
```bash
cd /var/www/TechTitans-AI
cp .env.example .env
nano .env
```
Ensure you set the production credentials:
- `DATABASE_URL` (production PostgreSQL link)
- `JWT_SECRET` (generate a strong key using `openssl rand -hex 32`)
- `EMAIL_PASS`, `EMAIL_USER`, `EMAIL_HOST` (smtp keys)
- `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`, `CLOUDINARY_CLOUD_NAME`

---

## Step 5: Build the Monorepo

Install dependencies, run database migrations (Prisma client generation), and bundle both backend and frontend applications:
```bash
# Install dependencies
pnpm install --no-frozen-lockfile

# Generate Prisma Client database structures
npx pnpm --filter @workspace/db generate

# Run TypeScript checks and compile both applications
pnpm run build
```

This generates:
- Frontend static assets inside `/var/www/TechTitans-AI/artifacts/techtitans-ai/dist`
- Backend server bundle inside `/var/www/TechTitans-AI/artifacts/api-server/dist`

---

## Step 6: Start Backend Process via PM2

Launch the Express API server under PM2 management using the pre-configured ecosystem file:
```bash
pm2 start ecosystem.config.cjs
```

Save the process list and configure PM2 to auto-start the server on VPS reboot:
```bash
pm2 save
pm2 startup
```
*(Copy and execute the command generated by `pm2 startup` in your terminal to complete the boot configuration).*

---

## Step 7: Configure Nginx Routing

Copy the pre-configured Nginx config template to Nginx's sites-available:
```bash
sudo cp nginx.conf /etc/nginx/sites-available/techtitans
```

Open and edit the Nginx configuration to update your domain name:
```bash
sudo nano /etc/nginx/sites-available/techtitans
```
*(Replace `yourdomain.com` and `www.yourdomain.com` with your actual domain name).*

Enable the server block configuration and restart Nginx:
```bash
# Enable site block
sudo ln -s /etc/nginx/sites-available/techtitans /etc/nginx/sites-enabled/

# Remove default Nginx site configuration
sudo rm /etc/nginx/sites-enabled/default

# Test config for syntax errors
sudo nginx -t

# Restart Nginx
sudo systemctl restart nginx
```

---

## Step 8: Configure SSL Security (Let's Encrypt HTTPS)

Install Certbot Nginx plugin and request an SSL certificate:
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```
Follow the interactive prompts to enable automatic HTTPS redirects. Certbot will automatically rewrite the Nginx site configuration to secure your traffic.

---

## Managing Your Deployment

To manage and inspect the running API backend:
- View live console logs: `pm2 logs`
- Check status: `pm2 status`
- Restart server: `pm2 restart techtitans-api-server`
- Stop server: `pm2 stop techtitans-api-server`
